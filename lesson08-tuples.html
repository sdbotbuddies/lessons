Quick Recap — What You Know

The collections you've already met 🧠

📋 List

An ordered box of items you can change any time — add, remove, swap.

pets = ["dog", "cat", "fish"]
pets.append("bird") # ✏️ change it!

📖 Dictionary

Pairs of key → value. Look things up by their label.

scores = {"Sam": 95}

🔤 Strings

Text you can slice, join, and shout with .upper().

name = "max"
name.upper()  # → "MAX"

Today's mission: add a 4th tool to your toolbox — the tuple — and learn when each one is the right choice.

The Big Idea

So… what IS a tuple? 🔒

A tuple is a list's twin — it holds a group of items in order. But there's ONE huge difference: once you create a tuple, you cannot change it. It's locked!

📋 A List uses square brackets

colors = ["red", "green", "blue"]

✏️ Like writing in pencil — you can always erase and rewrite.

🔒 A Tuple uses round parentheses

colors = ("red", "green", "blue")

🖊️ Like writing in permanent marker — once it's down, it stays.

🧠 Easy way to remember

List = Loose (can change). Tuple = Tight / locked (cannot change). Tuples even use the same shape of brackets as a hug: ( )!

🔤 The word "tuple"

It rhymes with "couple." A "couple" is 2 things; a tuple can be 2, 3, 5, or more — a fixed bunch of items locked together.

Why Tuples Matter

Why would we WANT something locked? 🤔

It sounds strange that "can't change" is a superpower — but locking data protects it from mistakes. Some things should never change!

🛡️

Protects important data

Your birthday is (month, day, year). It should NEVER accidentally change. A tuple keeps it safe.

⚡

Faster for the computer

Because tuples never change, Python can store them more efficiently than lists. Small speed boost!

📦

Groups that belong together

A point on a map is (x, y). The two numbers belong together — a tuple bundles them as ONE thing.

🔑

Key insight: Use a tuple when the group of items should stay exactly the way you made it — no surprises, no accidental edits.

How to Write Tuples 🐍

Tuple syntax in Python

# Make a tuple with parentheses birthday = (10, 25, 2013) # Read items by position (starts at 0!) print(birthday[0])  # → 10 print(birthday[2])  # → 2013 # Find how many items print(len(birthday)) # → 3
# Unpacking — give each item a name! point = (4, 7) x, y = point print(x)  # → 4 print(y)  # → 7
The Rules
1️⃣

Use round brackets ( ) and separate items with commas.

2️⃣

Read any item with square brackets and a number — counting from 0.

3️⃣

You cannot do tuple[0] = 99 — Python will give an error!

4️⃣

Unpacking lets you pull items out into named variables in one line.

⚠️ One sneaky rule: a tuple with a single item needs a comma: (5,) is a tuple, but (5) is just the number 5!

Tuples in Real Life

Where tuples show up around you 🌍

🗺️ Map coordinates

Your location on a map is locked together as (latitude, longitude). The two numbers always travel as a pair.

🎨 RGB colors

Every color on a screen is (red, green, blue). Hot pink is (255, 105, 180) — three numbers, one color.

📅 A date

A birthday like (10, 25, 2013) = month, day, year. It should never accidentally change!

📐 Screen size

Your phone screen is (width, height) in pixels, like (1080, 1920) — a fixed measurement.

🎵 A music note

A note can be stored as (name, beats) like ("C", 4) — the note and how long it plays.

🍕 A menu price

An item could be ("Pizza", 12) — the name and its price stuck together as one unit.

Side by Side

List vs Tuple vs Dictionary 📊

Feature📋 List🔒 Tuple📖 Dictionary
Brackets [ ] square ( ) round { } curly
Can you change it? ✅ Yes — add/remove/edit ❌ No — it's locked ✅ Yes — change values
How to look things up By position [0] By position [0] By key ["name"]
Example ["a","b","c"] ("a","b","c") {"pet":"dog"}
Best for... A collection that grows & changes 🛒 Fixed data that shouldn't change 🔒 Looking things up by a label 🏷️
🧭

The big difference: Lists and dictionaries can change. A tuple is the one that stays frozen exactly how you made it.

Decision Helper

Which one should I pick? 🤷

Ask yourself these quick questions:

🔒

"Should this data stay exactly the same forever?"

→ Use a TUPLE. Examples: a birthday, a color code, map coordinates.

📋

"Will I add, remove, or reorder items later?"

→ Use a LIST. Examples: a to-do list, players in a game, items in a cart.

📖

"Do I need to look things up by a name or label?"

→ Use a DICTIONARY. Examples: a phone book, scores by player name, settings.

⭐

Rule of thumb: If it should never change → tuple. If it grows → list. If it has labels → dictionary.
